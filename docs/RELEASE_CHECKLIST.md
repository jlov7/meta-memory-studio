# Release Checklist

Use this checklist before cutting a release tag.

## Execution mode

This repository is currently configured for a no-cost private workflow:
- GitHub Actions workflows are manual-only (`workflow_dispatch`).
- Day-to-day verification is expected locally via `make verify`.
- Full pre-release gate is expected locally via `make release-gate`.

## Pre-flight

- [ ] `git status` is clean on `main`.
- [ ] Run manual CI workflows in GitHub (CI + Security) and confirm green.
- [ ] `CHANGELOG.md` has release notes for the target version.
- [ ] Security workflow has no open high-confidence findings.
- [ ] Fresh DB backup exists (`make db-backup`).

## Verification

- [ ] Run `make release-gate` locally.
- [ ] Smoke-test demo import (`make demo`) and key UI paths.
- [ ] Validate export pack endpoint returns ZIP and redacts PII.
- [ ] Generate/update dependency manifest (`make sbom`).

## Release

- [ ] Merge release PR generated by Release Please.
- [ ] Confirm GitHub Release is created with tag and notes.
- [ ] Confirm package/version metadata in release notes matches code.
- [ ] Keep repo private in GitHub settings for this pre-release stage.

## Post-release

- [ ] Open follow-up issue for deferred improvements.
- [ ] Announce release notes and migration caveats (if any).
- [ ] Run retention cleanup dry-run and review output (`make cleanup-retention`).
