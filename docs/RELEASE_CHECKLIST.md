# Release Checklist

Use this checklist before cutting a release tag.

## Pre-flight

- [ ] `git status` is clean on `main`.
- [ ] CI on `main` is green.
- [ ] `CHANGELOG.md` has release notes for the target version.
- [ ] Security workflow has no open high-confidence findings.

## Verification

- [ ] Run `make verify` locally.
- [ ] Smoke-test demo import (`make demo`) and key UI paths.
- [ ] Validate export pack endpoint returns ZIP and redacts PII.

## Release

- [ ] Merge release PR generated by Release Please.
- [ ] Confirm GitHub Release is created with tag and notes.
- [ ] Confirm package/version metadata in release notes matches code.

## Post-release

- [ ] Open follow-up issue for deferred improvements.
- [ ] Announce release notes and migration caveats (if any).
