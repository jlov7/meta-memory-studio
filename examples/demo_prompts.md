# Demo prompts (stable)

## 1) Show memory contribution
“Explain what happened in this run and show whether memory helped. If it did, identify the single most helpful memory item and why.”

## 2) Find harmful memory
“List any memory items that decreased performance vs baseline and mark them for deprecation.”

## 3) Parameter grounding (tool-use)
“Given the user’s later request, what tool arguments should the agent have filled from memory?”

## 4) Drift signal
“Compare tool usage patterns across runs and flag anomalies.”
